One cargo bench run of core/benches/benchmark.rs — a star (one-factor-at-a-time) design over four axes: builder strategy, column layout, secondary index, and source. Each panel below isolates one axis against a fixed baseline; the raw table at the bottom holds every measured configuration.

WRITE

Serialize — ingest & encode

Building a Vortex array from 100,000 generated quads and writing it out. Baseline: sorted-stream builder, default layout, no secondary index.

QUERY

Match — pattern resolution

Resolving match_pattern and materializing the matching quads, across the six routing shapes the resolver branches on: S (primary binary search, 1 row), P/O (single-column index probe, 1–2k rows), PO (two-column prefix probe, 1k rows), G (no index covers graph — mask scan, 10k rows), and SPOG (every component bound, 1 row).

DECODE

Load & decode — read-back

Opening a store and decoding every quad back out (full scan), independent of any match pattern.
